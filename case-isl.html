Role

Solo builder — problem framing, model choice, and the full stack from webcam capture to trained classifier to the speaking front end. Locked the v1 scope, built it, trained it, merged it.

Stack

  • Python
  • TensorFlow
  • MediaPipe
  • LSTM
  • FastAPI

Context

Sign language is motion, not pictures

A deaf person signing "Good morning" isn't holding a pose — they're moving. Any recognizer that looks at single frames misses the thing that actually carries meaning. The product goal was simple to say and hard to do: point a webcam at someone signing and have the screen say the word out loud.

Indian Sign Language has far less tooling than ASL, so this started from the INCLUDE dataset (AI4Bharat) and word-level isolated signs — the realistic first rung before continuous signing.

Approach

Landmarks over pixels, sequence over snapshot

Rather than feed raw video into a heavy vision model, the pipeline extracts MediaPipe Holistic landmarks — hands, pose, and a face subset — and classifies the sequence of those landmarks with an LSTM. That keeps it light enough to run live on a laptop and focuses the model on the motion that distinguishes one sign from another.

  • Auto-segmentation: hands enter frame → buffer landmarks → hands pause → predict. A manual hold-to-sign mode is the fallback.
  • Speaking output: the recognized word renders as text and is spoken via the Web Speech API — the "translation" the user actually feels.
  • Tuned thresholds: the confidence gate sits at 0.3, not the naive 0.7 — at 9 classes the correct prediction often lands mid-confidence, and a high gate silently swallows it.
  • Local + private: a FastAPI WebSocket backend with a plain HTML/JS split-panel front end; runs entirely on localhost.

Architecture

The live loop

[ Webcam frames ]
      ↓     MediaPipe Holistic → hand + pose + face landmarks
[ Landmark buffer ]  // auto-segment on motion → pause
      ↓
[ LSTM classifier ]  // trained on INCLUDE, sequence in → sign out
      ↓
[ Word ]  ──▶  on-screen text  +  Web Speech API audio

v1 trained on the INCLUDE "Greetings" category — nine signs including Hello, Thank you, and the Good morning/afternoon/evening/night set — reaching 71% held-out accuracy, with the evening/night pair the weak classes and the other seven strong.

A dependency pin carries the whole thing: MediaPipe locked to 0.10.9, because the classic Holistic API was removed in later versions and newer builds drag in a NumPy that breaks TensorFlow. Getting the ML stack to simply coexist was half the battle.

Outcome

A working recognizer that speaks

v1 is built, trained, and merged: point the webcam, sign one of the trained greetings, and the screen shows the word and says it aloud in real time. It's the proof that the landmarks-plus-LSTM approach holds — and the base the class count grows on.

What's next

Scale the vocabulary, then the sentence

The near-term path is more classes from the ~50-sign INCLUDE set and per-user calibration to lift the weak pairs. The real horizon is continuous signing — recognizing a stream of signs as a sentence rather than isolated words — which is a genuinely harder sequence problem and the thing that makes it useful day to day.
